# Workload CSV Format

Benchmark workloads are replayed by [run_api_workload.py](/Users/kubaj/edutelligence/logos/tests/performance/run_api_workload.py).

## CSV Schema

Every workload CSV must use this header:

```text
request_id,arrival_offset,mode,priority,body_json
```

Field meanings:

- `request_id`
  Optional unique row identifier.
- `arrival_offset`
  Required millisecond offset from benchmark start.
- `mode`
  Optional. `interactive` or `batch`. Defaults to `interactive`.
- `priority`
  Optional. `low`, `mid`, or `high`. Defaults to `mid`.
- `body_json`
  Required JSON request body sent directly to `/v1/chat/completions`.

## Direct Model Routing

If `body_json` contains a valid `"model"` field, Logos uses direct model execution and skips classification.

Example:

```csv
request_id,arrival_offset,mode,priority,body_json
explicit-coder7-0001,0,interactive,high,"{""model"":""Qwen/Qwen2.5-Coder-7B-Instruct-AWQ"",""stream"":true,""messages"":[{""role"":""system"",""content"":""Follow the user instructions exactly.""},{""role"":""user"",""content"":""Patch a small Python handler and add two tests.""}]}"
```

## Current Benchmark Tree

- `explicit/10m/workload_explicit_local5_skewed_bursty_10m.csv`
- `explicit/60m/workload_explicit_local5_skewed_bursty_60m.csv`
- `resource/10m/workload_resource_local5_skewed_bursty_10m.csv`
- `resource/60m/workload_resource_local5_skewed_bursty_60m.csv`

These workloads intentionally:

- send normal chat completion requests
- use bursty arrival patterns
- keep a skewed but repeatable model-hit distribution

The `explicit/*` workloads:

- set `"model"` directly
- skip classification

The `resource/*` workloads:

- omit `"model"`
- rely on system-prompt steering so classification lands on the intended local models

## Generation

The explicit workloads are generated by [generate_explicit_model_workloads.py](/Users/kubaj/edutelligence/logos/tests/performance/generate_explicit_model_workloads.py).

Regenerate them with:

```bash
python3 tests/performance/generate_explicit_model_workloads.py
```

## Notes

- Additional CSV columns are ignored by the runner.
- Use Python `csv` + `json.dumps()` for generation so quoting stays valid.
- For resource-mode workloads, omit `"model"` from `body_json`. The current explicit benchmark set does not do that.
